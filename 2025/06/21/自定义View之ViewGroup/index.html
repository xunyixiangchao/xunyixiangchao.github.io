自定义View之ViewGroup

自定义View的基本方法：onMeasure()、onLayout()、onDraw()

  • 测量：onMeasure()决定View的大小；
  • 布局：onLayout()决定View在ViewGroup中的位置；
  • 绘制：onDraw()决定绘制这个View。

自定义View作为容器时，一般继承自ViewGroup，并且只需要实现测量和布局的流程，不太关心绘制。

问题

结合布局来看就发现了以下问题：

  1. 嵌套ScrollView后，子View的match_parent属性失效。
  2. 自定义布局添加padding值后，布局显示不全。
  3. 子View的margin属性没有生效。

问题一

​ 在ScrollView的源码中，measureChild或measureChildWithMargins时ScrollView直接将子View的高模式设置成了MeasureSpec.UNSPECIFIED

所以自定义View在getChildMeasureSpec时走的是UNSPECIFIED逻辑

父容器为UNSPECIFIED，子View为MATCH_PARENT时，子View的size就相当于WRAP_CONTENT自适应的效果。

如果我们自定义ViewGroup需要子View以MATCH_PARENT显示的话。我们需要对自定义ViewGroup进行高模式的EXACTLY设定

面试题

<1>onMeasure(widthMeasureSpec, heightMeasureSpec) 回调的两个参数是什么?

答： 是「模式」与「尺寸」的综合度量值。 格式是: xx yyyyyy….yyyy 共 32 位, 高 2 位代表模式, 后 30 位代表

尺寸。其中「模式」有 UNSPECIFIED(00), EXACTLY(01), AT_MOST(11)。

  • xml 设置为具体某个 dp 时, 模式为 EXCATLY(精确的) -> 01
  • xml 设置为 wrap_content/match_parent 时, 模式为 AT_MOST(最大) -> 11
  • UNSPECIFIED 一般用于 ScrollView, 由 ScrollView 传递给其子 View, 子 View 根据该模式自行确定大小 -> 00

<2>MeasureSpec.getSize()/MeasureSpec.getMode() 的作用是什么?

答： getSize()是从 Spec 中获取尺寸, getMode()是从 Spec 中获取模式。

<3> 为何 Spec 中, 「模式」放在首 2 位?

答：因为目前设备的尺寸一般不会超过 32 位, 为了同时用一串数字表达模式和尺寸, 把模式放在高 2 位表达会比较

方便, 通过移位操作即可获取到「模式」和「尺寸」

<4>getChildMeasureSpec()的作用是什么?

答：是用于确定子控件的「模式」和「尺寸」, getChildMeasureSpec 里有一套算法, 根据父布局的模式以及业务

需求计算所得宽高, 传递给 getChildMeasureSpec 方法进行综合计算后, 返回子控件实际所得的宽高尺寸.

该方法综合父控件的 Spec 来测量子控件所得的大小, 如下表格所示:

<5>child.measure(measureWidthSpec,measureHeightSpec) 的作用是什么? 传递进去的参数作用是什么?

答: measure 方法就是把业务所需的模式和尺寸传递给系统进行真正的测量. 其传递的参数就是业务计算所得的模

式和尺寸.每调用一次 measure 方法, 如果控件中还有子控件, 则会触发其内部子控件的 onMeasure 方法.

<6> child.measure()测量后可以立即得到测量后的宽高吗?

答: 肯定可以, 因为 measure 的目的就是把数据交付给系统进行操作, 系统测量后数据就存入了 measureCache

中。

<7>setMeasureDimension(measureWidth, measureHeight) 的作用是什么? 其传递的参数是什么?

答: setMeasureDimension()方法的作用就是由业务需求以及考虑其内部子控件的影响因素后, 最终确定要其控件

本身设置的大小, 其传递的参数就是真正决定其控件本身的大小的值。这两个值设置多少, 控件在屏幕就显示多

少。

<8>getMeasureWidth()和 getWidth()的区别?

答: getMeasureWidth()在其控件本身的onMeasure回调中可直接得到测量值; 或者其内部的子控件调用measure()

方法后, 子控件调用 getMeasureWidth()可以得到刚刚测量的值。getWidth()是能过onLayout方法后设置的左上右下后通过right-left获得的值。

<9> MeasureSpec. makeMeasureSpec(size, mode) 的作用是什么?

答: 子控件进行 measure 时, 要求传入参数 widthMeasureSpec, heightMeasureSpec. 这两个参数一般根据父控

件对其的尺寸因素影响最终计算所得. 但有时候一些比较简单的需求, 能确切知道子控件的大小以及模式时, 就可以

采用 makeMeasureSpec()这种方式来创建 Spec, 然后再给子控件 measure()的时候传递进去。

<10> 为什么 onMeasure 有时候会调用多次?

答: 调用多少次是由其所在的父控件的算法所决定的。

如 LinearLayout 会测量 2 次; FrameLayout 会测量 2 次; RelativeLayout 会测量 4 次;

<11> onLayout(l,t,r,b) 回调的参数是什么?

答: 是其所在的父控件在调用layout后返回给其的位置坐标值。 同理, 当在onLayout中对其内部的子控件进行调用

layout 方法的调用, 也会触发子控件的 onLayout 方法. 在 onLayout 后, 控件就真正的布局到了屏幕上。
